# Bootstrap confidence intervals on v0 vs v1 metric deltas

Every v0 vs v1 metric delta reported in the writeup will include a 95% bootstrap confidence interval, not just a point estimate. The procedure: take the per-case scores stored in Postgres for each version, draw 1000 resamples with replacement, compute the v1 minus v0 delta on each resample, and report the 2.5% and 97.5% percentiles as the CI bounds. A CI that excludes zero indicates a robust improvement; a CI that crosses zero indicates the headline delta is within the noise band of our small-N test set and the writeup must say so. This is pure post-processing on stored data: no extra LLM calls, no API cost, no additional runtime beyond milliseconds in numpy.

We rejected classical hypothesis tests (paired t-test, Wilcoxon) because eval scores are bounded between 0 and 1, our N is small (~24 visible plus 6 held-out), and p-values invite a false sense of rigor that does not help readers calibrate trust in the headline number. We also rejected reporting raw deltas without uncertainty quantification because it cannot distinguish a real win from a noise-driven swing, which is exactly the question an honest writeup must answer at this sample size. Bootstrap CIs are robust to the non-normal distribution of bounded eval scores, give directly interpretable bounds, and signal statistical literacy to the audience without resorting to p-value theater.
